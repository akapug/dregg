REFLECTIVE JS SCRIPTING is a deos-native scripting surface where you write JavaScript and the objects you touch ARE live handles into the running image — cells, caps, the ledger, affordances. It is "cap-gated Pharo, not omniscient Pharo": reflection is cap-bounded and attested (you observe only what you can produce a witness for), and interaction is production-under-non-forgeability (every fired affordance is a verified turn). The engine is real SpiderMonkey via deos-js (the mozjs_sys 140.x static archive — evaluate_script("40+2")=42, no stub), bound over the gpui-free deos-reflect substrate. (Design: docs/deos/SCRIPTING-AND-DISTRIBUTED-DOM.md.)

What it shows (proven by running): from JS you deos.world.cells() to crawl the ledger and deos.world.ocap() for the capability web; deos.cell(id).reflect() reads a cell's four substances (value · authority/c-list · state · evidence) and .field(k) a single field; .as(viewer) is a real per-viewer FRUSTUM (.canObserve / .reflect) — not a filter but a cap-bounded crawl where an unreachable cell is ABSENT, never forged. The crawl was validated by reading a cell's balance substance and confirming the frustum is cap-bounded: a viewer observes itself, but a stranger is absent and its reflect()===null. The drive half (deos.applet({affordances})) mints one cell on an embedded DreggEngine and each affordance call commits a REAL cap-gated verified turn — four fires leave four TurnReceipts — while a view-state set (app.view.set(...)) leaves NO receipt, and a Proof affordance held by a Signature driver is REFUSED (anti-ghost).

What it proves: the two halves are kept apart on purpose. Reflection commits NO turns (crawling is a READ — you cannot observe past your authority, and reads are attested + non-omittable); interaction is a production discharged through the kernel's gateOK, fail-closed, leaving a receipt. So a button you may press is a proposition you can prove, and a stranger's cell is genuinely invisible rather than merely hidden. Reproduce: cd deos-js && cargo test (the js_drives_substance suite — the drive and crawl slices both green on real mozjs). The binding is the artifact and is language-agnostic; JS is first only because mozjs is already in-tree. (This surface carries no committed screenshot; the demonstration is the test suite.)
