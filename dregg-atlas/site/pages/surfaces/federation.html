A real two-node dregg-node federation over QUIC gossip + blocklace consensus: a faucet turn on A gossips to B and both DAGs converge byte-identically to consensus-attested finality. (No committed PNG — the proof is the runbook + the named net/node tests.)

The FEDERATION surface is a real multi-node dregg-node deployment over QUIC gossip + blocklace consensus: two nodes sharing a 2-validator genesis form an n=2 committee, and a turn submitted on one node finalizes on the other. Nothing here is a mock — both nodes run the verified Lean producer, exchange real Ed25519-signed blocks, and converge to a byte-identical DAG.

What it shows: the committee is established by a SHARED genesis.json carrying both validators' public keys (the same committee → the same federation_id → the same participants). With both nodes running --federation-mode full and cross-pointing gossip ports, a faucet turn on node A gossips to B; within ~5s both /api/blocklace/blocks return the IDENTICAL 6-block DAG, latest_height=1 on both (the turn block super-ratified cross-node), dregg_consensus_attested_total 1 on BOTH (consensus-wide Attested), and the recipient's balance = 5000 on both nodes — the effect applied through consensus finalization, not locally.

Robustness: each node runs a peer reconnect prober (blocklace_sync::spawn_peer_prober) so a peer DOWN at boot (or one that drops and returns) still converges without an operator restart, and a bounded 3s initial dial means a down peer no longer stalls startup. Discovery is authenticated gossip-of-peers: a node shares cryptographically-verified peer-identity → listen address bindings, and a receiver accepts an address ONLY for a key already in its genesis-trusted committee — so the mesh forms transitively from a single seed while a forged Sybil binding is rejected.

How you reproduce it: the full staged recipe (mint a 2-validator genesis, build two data dirs, run both nodes, faucet on A) is in docs/deos/DEV-NODE-RUNBOOK.md; the reconnect / discovery / quorum paths are pinned by named net/node tests (late_join_prober, gossip_of_peers_transitive, gossip_of_peers_accepts_committee_rejects_forged, quorum_crossing_is_reported_on_whichever_vote_is_second).
