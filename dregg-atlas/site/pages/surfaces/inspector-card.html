The REFLECTIVE INSPECTOR CARD is the Inspect-mode main surface reborn as a live deos-js CARD — the rung-2 reflective turn of the fully-reflective-cockpit campaign (cockpit/mod.rs::InspectorCardMount, card_pane.rs; gated #[cfg(all(feature = "dev-surfaces", feature = "card-pane"))]). Where the plain INSPECTOR tab draws the focused cell's faces with hand-written gpui, this surface *generates* a CardPane view-tree from those same moldable faces and hosts it as a card the image renders — the surface that inspects the image is itself an object inside the image.

What it shows: the focused cell's moldable presentation faces, projected into a CardPane entity built over the LIVE World and held in the cockpit's inspector_card mount alongside a shared attached applet the rendered widgets drive. The card is LAZY — None until the Inspect surface first paints, then built by ensure_inspector_card, and rebuilt whenever the focus moves (the mount records the CellId the view-tree was generated for). On the gpui-free / card-pane-off build the mount is absent and the Rust moldable inspector remains the surface.

How you use it: it behaves like the inspector — you read the focused cell's faces — but every affordance button on the card fires a REAL verified turn on the operator's real cell through the embedded executor, and a bind re-reads the post-state, so acting on the card moves the live ledger and the card reflects it. This is the keystone of "v2 = a frame you can rewrite while it runs": the inspector is no longer privileged gpui code but a card authored in the same deos-js substrate the user can reshape. (Baked here via the inspector tab — when card-pane is on, the card IS the Inspect surface.)
