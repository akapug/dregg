# lean-orphans-allow.txt — modules under metatheory/Dregg2/ that are DELIBERATELY
# not reachable from any default lake target (Dregg2/Metatheory/Polis/Market/Bfv).
#
# WHY THIS FILE EXISTS
#   The `Dregg2` lean_lib has no `globs`, so `lake build` compiles only the modules
#   transitively imported from metatheory/Dregg2.lean. A new metatheory/Dregg2/**.lean
#   that nothing imports builds green under `lake env lean <file>`, passes its own
#   #assert_all_clean / #assert_axioms / #guard, and is covered by NOTHING in CI — a
#   downstream regression can silently break it and no gate goes red. That hole recurred
#   (StepBridge, AciNormal, AciComplete) and was each time caught only by a human reviewer.
#
#   scripts/check-lean-orphans.sh FAILS if any Dregg2/**.lean is neither reachable nor listed
#   here. So a NEW orphan reds CI and forces a decision: register it in Dregg2.lean, or add
#   it here WITH A REASON. Registering a listed module and NOT removing it here is ALSO a
#   failure (stale entry) — the allowlist cannot silently outlive the exclusion it records.
#
# FORMAT  one dotted module per line, optional `# reason` after it. `#`-only lines and blanks
#         are comments. Every listed module MUST exist as a file and MUST be a genuine orphan.
#
# These entries are the CURRENT (2026-07-18) orphan census — WIP from this and other
# terminals. As each cluster's work lands, register the finished modules in Dregg2.lean and
# delete them from here; the staleness check will remind you.
