# Dregg Desktop OS — Architecture Proposals

*The three independent architecture proposals (distinct lenses) from the design workflow's Architect phase. All three self-scored 9/10 and converged on the same R0 first slice — that convergence is itself the signal. The synthesis is in [DREGG-DESKTOP-OS.md](../DREGG-DESKTOP-OS.md).*

## Lens: Fuchsia/Zircon, read STRICTLY through dregg's own grain: Zircon-handle ↦ firmament `Capability{target,rights}`; Zircon-object ↦ dregg cell; ViewRef ↦ `CellId` (content-addressed, already unforgeable); Scenic/Flatland View↦Viewport ↦ a surface-cell + a `Target::Surface{cell}` cap reaching it; the eventpair token handshake ↦ `Spec.Authority.Introduce` (the SAME 4 premises: connectivity + holds-parent + non-amplifying + consent); FIDL object IPC ↦ a turn through the executor (or, off-the-wire local, a Microkit pp_call); the Component Framework ↦ factory-birth (`FactoryDescriptor.allowed_cap_templates` ↦ Authority.Mint contract). The Zircon DELTA that makes this dregg-not-a-clone: a Zircon handle is a kernel-table integer with a fixed rights bitmask checked by goodwill of a 2M-LOC kernel; a firmament handle is the SAME abstraction whose attenuation is `is_attenuation` (a proven lattice law) and whose evolution a light client can verify it cannot be fooled about (`unfoolability_guarantee`). So: keep Zircon's object-and-handle SHAPE and its windowing/composition SPLIT (kernel owns the view-tree-as-state, compositor is a dumb renderer); replace its trust basis with dregg's verified ocap + n-parametrized distance. A window is not "an object that has a capability" — it IS a capability (a `(Target::Surface(cell), rights=region∥effect-facet)` point on the one gradation). The compositor is the net-PD/persist-PD's third sibling: sole holder of the framebuffer+input device caps, zero app logic.  *(self-score 9/10)*

> dregg-OS = the firmament's one (target,rights) handle carried all the way to pixels and HID: a window IS a `Target::Surface{cell}` capability, the compositor is the third device-holding PD sibling, and the whole stack runs the SAME cap-first code on a mac/linux semihost emulator today and on real seL4 later — Zircon's object/handle+scene-graph shape on dregg's verified-ocap trust basis.

**Layers (bottom→top)**

1. L0 — HARDWARE / virtio. aarch64-QEMU-virt is the primary rung (riscv64 proven for M0); a virtio-gpu + virtio-input device set joins the already-probing virtio-net. seL4 owns nothing here it hasn't been given a device cap for.
2. L1 — KERNEL. Real seL4 (M0/M1/M2/M-STARK already boot in QEMU via the native-macOS Microkit 2.2.0 toolchain) OR the semihost `EmulatedKernel` (a host-native faithful model of the seL4 cap/IPC surface: CSpaces with a mint/revoke derivation tree, synchronous Endpoints, badge-OR Notifications, Untyped+Retype). The SAME PD source compiles against either, selected by cfg — this is the portability fulcrum.
3. L2 — DEVICE-HOLDING DRIVER PDs (the firmament topology, FIRMAMENT.md §2). Each is the SOLE holder of one device cap + its virtio-mmio region + DMA paddr, cloned from the proven net driver-PD: net-PD (NIC, boots), persist-PD (disk, redb-over-block), gpu-driver-PD (framebuffer/GPU — NEW), input-driver-PD (HID — NEW). Plus a libvmm Linux compat-VM as a big app-PD for the migration fulcrum.
4. L3 — THE VERIFIED EXECUTOR-PD (the firmament's heart). The Lean `execFullForestG` closure (already an aarch64-musl ELF running a real turn) hosted in a `sel4-root-task-with-std` PD over `sel4-musl`; on the semihost it links the ordinary host Lean runtime so the emulator has a REAL verified heart with no libuv-excision blocker. Every authority decision, every cap mint/attenuate/revoke, IS a turn here.
5. L4 — THE FIRMAMENT ROUTER (`sel4/dregg-firmament`, BUILT). The ONE `Capability{target,rights}` handle over three backings: `Local{slot}`→seL4 CNode (or emulated CSpace), `Distributed{cell}`→executor-turn→net, and the NEW `Surface{cell}`→a compositor present()-turn. `is_attenuation` (granted⊆held) is the single attenuation law at all three; `Bounds` carries the n-parametrized honest bounds (n=1 collapse = immediate revoke + synchronous commit).
6. L5 — THE COMPOSITOR-PD (the third device sibling; Nitpicker/EWS-minimal TCB). Sole holder of the framebuffer + input device caps. State = the scene graph as a verified dregg cell: an ordered list of surfaces (owningCellId, regionRect, contentDigest, sourceStateRoot, zLayer, focusFlag). present()/embed()/grant-input()/revoke are capability-gated verbs; the scene invariants (non-overlap, label=owner, at-most-one-focus, no-cross-read) are anti-ghost teeth proved with the existing VerificationToolkit AppSpec. Holds NO app logic, NO widget toolkit, NO placement policy.
7. L6 — THE SHELL + WINDOW-MANAGER CELLS (untrusted, client-side handling per Nitpicker/EWS). A window-manager is a compositor-client-cell that is ALSO a compositor to the apps it frames (Genode recursive stacking, free from cells-holding-caps-to-cells). Focus-routing within a subtree is a delegated cap (Scenic hierarchical focus). The trusted-path SAK PD + reserved chrome live just above as the one non-cell trust anchor.
8. L7 — APP CELLS + WEB SURFACES. Every app is a dregg cell; its window is a child surface-cell it birthed via a surface-factory cap; a servo WebView binds 1:1 to a surface-cell via OffscreenRenderingContext (the DarpaBrowser confinement story — a hostile page holds only its surface cap). starbridge-v2's gpui face is the native-now compositor; the seL4 compositor-PD is the verified-later destination the same surface model scales to.

**Semihost strategy**

ONE PD source tree, two backends by cfg — the UML `um`-arch / gVisor-platform pattern applied to the rust-sel4 API. The contract (the "port") is the `sel4-microkit` surface the dregg PDs ALREADY code against: `#[protection_domain]`, the `Handler` trait (notified/protected/fault), `Channel` (notify/pp_call/irq_ack), `MessageInfo`+IPC-buffer, `memory_region_symbol!`. Build a NEW standalone crate `sel4/semihost/` (its own workspace, like dregg-firmament) with: (A) `semihost-kernel` — PROMOTE `dregg-firmament::LocalBacking` (already a real CSpace slot-table with a mint/revoke derivation tree reusing `dregg_cell::is_attenuation`) into a full `EmulatedKernel` by adding a synchronous `Endpoint` (Call parks on a condvar until Recv) and a badge-OR `Notification`; std-only, runs on mac+linux, no root, no QEMU, no nightly. (B) a `sel4-microkit`-API facade with IDENTICAL upstream signatures, std-backed, where `Channel::notify`→signal an emulated Notification, `Channel::pp_call`→an emulated Endpoint Call, the run-loop blocks on the kernel's Recv. PDs are host THREADS v0 (fast, but shared address space ⇒ "no ambient authority" is API-by-construction not MMU-enforced — the ONE honest non-fidelity, labeled exactly as UML's traced-thread→SKAS gap and closed by the v1 process-backed-PD upgrade + the real-seL4 target). `memory_region_symbol!`→host shared buffers so the net-client's smoltcp-over-shared-ring code runs unchanged. KEYSTONE TEST: take `m0-hello` + a 2-PD notify slice + (the payoff) the surface loop, and `cargo test` the SAME source on the host that today only boots under QEMU; assert the notified handler saw the badge, a pp_call round-trips a MessageInfo, a CNode mint+revoke enforces granted⊆held, AND a present()-turn paints + a revoke darkens synchronously. The emulator advertises `Bounds::LOCAL` and these are GENUINELY real on the host (a host thread's revoke IS synchronous) — a faithful n=1 firmament, not a lossy mock. Gate it with a shared integration test that later boots the SAME PDs on real seL4 so the same-code claim is continuously checked. Honest determinism win: the emulator is the natural place to make the houyhnhnm clock/RNG a recorded, replayable cap-input for free.

**Compositor model**

THE COMPOSITOR-PD = the net-PD/persist-PD's THIRD device-holding sibling, sole holder of the framebuffer/GPU + input device caps, structurally identical to how persist-PD solely holds the disk cap. Minimal TCB (Nitpicker/EWS ~1.5K-LOC discipline): own the screen + input devices, route pixels/events BY CAPABILITY, render the trusted chrome — NO app logic, NO widget toolkit, NO placement policy (those live in untrusted shell/app cells). MODEL THE COMPOSITOR AS ONE VERIFIED DREGG CELL whose state IS the scene graph: an ordered list of surfaces = (owningCellId, regionRect, contentDigest, sourceStateRoot, zLayer, focusFlag). Compositing is a TURN: a cell submits `present(region, contentDigest @ myStateRoot)` against the compositor cell, and the executor's caveat gate enforces the security invariants AS anti-ghost teeth proved with the EXISTING VerificationToolkit AppSpec (zero new axioms, runnable in starbridge-v2's embedded executor + the semihost TODAY): T1 NON-OVERLAP — a cell writes only regions its cap authorizes; region-set is Rights on the SAME is_attenuation lattice (granted⊆held on screen rectangles); an overpaint is UNSAT. T2 LABEL-BINDING — every surface's rendered label is a function of its owningCellId + sourceStateRoot (read by the compositor from cell state, NOT the app); a label≠owner is UNSAT (Nitpicker's floating label UPGRADED from a server courtesy to a verified state-root binding a light client can independently check — richer than Qubes' flat color border because dregg authority is a structured provenance lattice: which factory minted it, sovereign-vs-hosted, attenuation depth). T3 FOCUS-EXCLUSIVITY — at-most-one focusFlag; input routes only to the focused surface's input-cap holder; delivering input to a non-focused cell, or two focus flags, is UNSAT (EROS traceability-of-volition as a state invariant). T4 NO-INFERENCE — present() reads only its own region's prior contents. FIVE cap-confined verbs, all generative-disciplined: CREATE-SURFACE (authorized by a surface-factory cap; the factory CONTRACT = Authority.Mint's `allowed_cap_templates` bounds what surface/input rights the cell may ever hold — Arcan typed-archetype, verified); PRESENT(surface,dirty-rect) (client frame-cap mapped in by an seL4 frame-cap grant — Qubes zero-copy, n=1 collapse; compositor never copies, only maps+composites; requires a write-cap on the surface); EMBED(parent,child) (a surface-tree edge authorized ONLY by `Spec.Authority.Introduce` — Fuchsia's non-clonable token handshake IS dregg's introduce; the surface tree is decoupled from the cell-ownership tree per RFC-0147); GRANT-INPUT/GRANT-CAPTURE (the explicit attenuable cap Wayland leaves as an ungoverned 'privileged client' bit — in dregg a global screenshot cap simply does NOT exist unless minted); REVOKE (drops the surface cap; n=1 immediate via seL4_CNode_Revoke, or distributedly the group-key epoch lift). TRUSTED PATH = a tiny SAK PD holding the SOLE input cap + the SOLE top-zLayer surface cap; a kernel-routed key combo (semihost: a host chord) summons an unspoofable overlay reading owner+state-root straight from compositor-cell state — promoting starbridge-v2's cipherclerk/Cmd-K palette to the trust anchor. Because the SAK surface is at a zLayer outside every cell's region grant (T1), no cell can overpaint it. ATTESTED USER-VOLITION: when a gesture in a trusted surface authorizes a value-moving turn ('sign this'), the compositor-PD emits a SIGNED input-receipt the executor-PD requires as a turn premise (EWS volition via dregg's existing attestation/receipt machinery, mirroring the net-PD's Ed25519 pre-check) — the receipt commits to (surface CellId + gesture + nonce + which turn-field) to defeat replay/confusion. INPUT vs OUTPUT throughput tier (the real concession): focus GRANTS are real receipted turns (rare, security-relevant, auditable WHY this window got my keystroke); the per-event 1000Hz stream flows on a faster cap-mediated shared-ring the input-cap authorizes, checkpointed into receipts at the cadence the proving-modality dial (task #169) sets. The GPU/framebuffer TCB decision: v0 = a SOFTWARE compositor cell (CPU blit of client frame-caps, EROS/Nitpicker path) where T1-T4 are REAL; GPU acceleration is a SEPARATE untrusted render-PD whose output is itself a frame-cap the compositor composites (preserves the trust model, bounds performance); the verified-graphics frontier (frame-attestation binding scanout to contentDigest, IOMMU/DMA confinement) is named as severe-problems-with-closure-lanes — the IOMMU gap is the graphics crypto-floor-equivalent, labeled exactly as sDDF flags it, never a wall.

**App / object model**

APP = CELL (exists today in starbridge-v2's embedded World): its authority is its c-list of CapabilityRefs, its observable state is its cell-forest, its history is its receipt chain — no new abstraction. WINDOW = a CHILD SURFACE-CELL addressed by a `Target::Surface{cell}` cap, mirroring Fuchsia View/Viewport/ViewRef with ZERO new kernel verbs: View := a surface-cell birthed via a surface FactoryDescriptor (state slots = contentDigest [BLAKE3 root of pixel buffer OR retained scene-graph/DOM], intrinsic-size, damage/epoch counter; born ZERO-balance + factory-validated, so it's a provenance-carrying object not an ambient handle); ViewRef := the surface-cell's `CellId` (already unforgeable, content-addressed via derive_raw, freely duplicable, cross-process — no new token type); Viewport := a CapabilityRef the parent app-cell HOLDS reaching the child surface-cell, attenuated to a SURFACE FACET via `CapabilityRef.allowed_effects` (the EffectMask, already proven narrow-only) — surface-relevant bits go in the FREE space (EffectMask bits 24-31 are unused; 0-23 are taken) as EFFECT_SURFACE_PAINT/RECEIVE_INPUT/RESIZE_HINT/CAPTURE, and a surface facet FORBIDS {seal,destroy,set-permissions,transfer} so a compositor holding only a Viewport-cap can render+route-input but cannot tamper. The token-PAIR handshake := a GrantCapability turn whose granted⊆held gate IS Fuchsia's 'only matching-token holders link' — strictly STRONGER than a kernel eventpair because attenuable AND its provenance is on the receipt chain. THE VIEW TREE = THE OCAP GRAPH (just project the World's CapabilityGranted/CapabilityRevoked dynamics edges). FOCUS IS A CAPABILITY: the compositor-cell grants a short-lived revocable receive-input facet to exactly one surface-cell; focus change = a revoke+grant turn, so 'only the focused view gets input' is ocap-enforced and the focus history is auditable on-chain. REACTIVITY: a SetField on a surface-cell's content/damage slot emits a `SurfaceDamaged` WorldEvent carrying the new contentDigest + damage rect; the compositor's existing per-frame `dynamics.since(cursor)` poll re-renders that surface — state-to-pixels is a PURE FUNCTION on the verified dynamics stream, so the whole desktop is REPLAYABLE + TIME-TRAVELABLE (starbridge-v2's replay/debug panels get surfaces for free). DISTRIBUTED + WEB-FORWARD = the firmament router UNCHANGED: a surface-cap is a firmament Capability{target,rights}; LOCAL target → compositor renders directly (n=1 immediate); REMOTE surface-cell → FirmamentRouter resolves via executor→net and the SAME window renders a remote cell's surface (the fluid reach-out applied to pixels). Two remote transport modes, both grounded in existing mechanism: DETERMINISTIC content (retained scene-graph/DOM/verified app) ships STATE not pixels (the Croquet lesson) — the window subscribes to the remote surface-cell's state-commitment + re-derives pixels locally, and the state-root tooth makes the embedded remote surface SELF-ATTESTING (dregg's NOVEL answer to Arcan's 'no visual identity to safely forward'); OPAQUE content (video, webgpu canvas) ships a contentDigest in state + bytes over the net-PD shared-ring with the commitment binding bytes to the receipt chain. Surface migration across the wire = Arcan DEVICEHINT reconnection expressed as the firmament's n=1→n>1 slide: the surface-cap is unchanged, only its Bounds relax. THE POWERBOX (CapDesk): the trusted compositor-cell IS the powerbox — the user picks a file/peer in the trusted UI and the compositor MINTS a fresh attenuated CapabilityRef into the app-cell's c-list via a real grant turn — how a confined app gets NEW authority without ambient grant. Naming/discovery of a remote surface to embed = dregg-directory (human-readable name → surface sturdy-ref with expiry/dispute), the unforgeable+attenuable+revocable improvement over a Plan 9 9P fid.

**Migration path**

BEST-ON-MAC/LINUX-NOW → seL4-NATIVE-LATER, three concurrent fronts that converge, NEVER a flag-day. FRONT 1 (today, the day-to-day fulcrum): starbridge-v2's gpui face IS the native-now compositor; build the surface model THERE first (it already embeds the verified executor + World + a surface.rs that is currently a shell-internal secret token — the work is to UNIFY that toy into the real firmament `Target::Surface` cap). The semihost emulator runs the SAME cap-first PD code natively on mac/linux with a REAL verified executor heart (no Lean-port blocker), advertising honest Bounds::LOCAL. FRONT 2 (the seL4 substrate ladder, additive QEMU boots): Stage A — host the executor-PD (`sel4-root-task-with-std` + `sel4-musl`, the single highest-value seL4 unblock, swapping the stub in dregg.system); Stage B — close the net edge to a full TCP→turn round-trip (the n>1 end real end-to-end); Stage C — gpu-driver-PD + input-driver-PD cloned from the proven net driver (virtio-drivers 0.13.0 already pins VirtIOGpu/VirtIOInput with the sel4 hal-impl — a WELD not a build; ramfb is the de-risked fallback display); Stage D — the compositor-PD multiplexer (banscii artist's region_in→composite→region_out→flush generalized to many surfaces + the firmament LocalBacking mint/revoke for window lifecycle); Stage E — the bootable `dregg.system` desktop (`make run-desktop`: a verified turn's result appears on-screen, driven by real keyboard/mouse). FRONT 3 (the SERVICE-MIGRATION fulcrum, for legacy reach): a libvmm Linux compat-VM boots as a big app-PD (zero new toolchain, sidesteps the executor blocker); each device/service is lifted one-by-one from in-VM to native dregg PD via the sDDF spine (the net-PD is ALREADY an sDDF driver/client pair — generalize it: driver→virtualiser→{native client | VM virtio backend}); extraction order easiest-first NETWORK→TIMER→SERIAL→BLOCK→INPUT→DISPLAY-last; a dbus→turn shim (a badged-endpoint bus-broker PD, Genode pattern) turns Linux service calls into real mandate-gated dregg turns so a Linux app calling org.dregg.Storage.Put runs a verified turn — the SAME fluid local↔distributed reach-out extended to the Linux/dregg seam, with the well-known-name table backed by the rbg DirectoryCell as a c-list. Each step is staged-additive-then-cutover (the MINTED pattern): the native PD lands BESIDE the VM driver, the virtualiser routes to it, the guest driver is disabled. The honest framing: display+HID are INHERENTLY-LOCAL (n=1) seats — the desktop is the n=1 collapse made visible — while net is the only seat that slides along n; the same binaries scale from the semihost to the seL4 image, the verified destination.

**Risks**

- GPU/framebuffer TCB (Nitpicker's unsolved open problem, inherited): software CPU-composition keeps the compositor-PD tiny+verifiable but bounds performance; webgpu/servo/wgpu-Vulkan drags a huge GPU driver into/beside the TCB. The frame-attestation last-hop (binding scanout to contentDigest) needs either a verified display driver that hashes what it scans out, or trust-the-GPU-with-an-IOMMU-cap — and seL4 does NOT yet verify SystemMMU translation, so 'T1 holds against a malicious DMA-capable driver' is currently a NAMED ASSUMPTION (the graphics crypto-floor), not a theorem. Mitigation: v0 software compositor where T1-T4 are real; GPU as a separate untrusted render-PD whose output is a composited frame-cap; label the IOMMU gap as sDDF does, drive it to one semantics, never a wall.
- The executor-PD seL4 blocker is real and gates the seL4 FRONT (Stage A onward): `sel4-musl` is experimental + single-file and may not implement every syscall the Lean leanrt GC+malloc+TLS bootstrap issues (futex semantics, mmap flags for the GC arena). Mitigation: a strace of the already-running aarch64-musl ELF enumerates the exact set before the PD link; the semihost FRONT and the service-migration FRONT (pure sDDF device-sharing, no turns) both deliver value BEFORE the executor-PD lands, so it is not on the critical path to a demoable desktop.
- Semihost thread-model fidelity gap: host threads share an address space, so 'no ambient authority' is API-by-construction not MMU-enforced — a buggy/hostile in-process PD can scribble another's memory the emulator cannot stop. This is the SAME gap UML's traced-thread→SKAS faced. Risk if not labeled honestly: the emulator's Bounds say n=1-local but isolation is weaker than the seL4 mechanism. Mitigation: advertise it explicitly (verified MODEL on the emulator, verified MECHANISM on seL4); the v1 process-backed-PD upgrade (shm_open/mmap regions) + opaque generation-tagged cap handles (host RAM has no CHERI tag bits) close it.
- Input-event throughput vs receipted-turn purity: a 1000Hz input stream cannot each be a consensus-finalized turn. The two-tier split (focus GRANTS = real turns; per-event stream = a faster cap-mediated shared-ring checkpointed into receipts) is sound but the checkpoint cadence is an open dial (task #169) — set it wrong and either the receipt chain bloats or volition-attestation freshness weakens.
- Remote-surface latency under n>1: a live remote window may render ONE stale frame AFTER a revoke (the screen-share-already-stopped-but-one-more-frame-leaked window) because n>1 revocation is eventual (epoch-lift propagation). For value-revealing surfaces this needs a stricter freshness caveat (the depth-parameterized non-revocation freshness scale, task #150) so a revoked remote surface cannot paint even one frame; for n=1 (the desktop's common case) revoke is immediate and this risk is absent.
- Servo's heft (servo-script/servo-layout/SpiderMonkey + servo-media, rc-quality) for v1 web-forward: may be premature vs a thinner native webview or a wgpu-canvas+minimal-HTML until the desktop needs full web fidelity; the servo↔compositor texture seam (GL-offscreen blit vs Vulkan interop vs waiting on servo's wgpu renderer #37149) picks the integration cost. Mitigation: the SurfaceCell seam is renderer-agnostic — a WebView is just one surface-cell backing — so this is a deferrable backend choice, not an architectural commitment.
- gpui is single-threaded with an Rc<RefCell<World>> model — a multi-surface/multi-app/multi-PD compositor may not fit gpui's app/window model cleanly (is gpui the compositor or a client that submits surface-cells?). Mitigation: native-now gpui draws the demo; the seL4 compositor-PD is a separate minimal multiplexer (banscii-shaped), so gpui's threading model does not constrain the verified destination.

## Lens: DREGG-FIRST-PRINCIPLES: the desktop is the firmament made visual and interactive. A window is not a new abstraction — it is a `Capability { target: Surface(cell), rights }` on the SAME `(target, rights)` handle that today resolves `Local{slot}` (seL4 syscall) or `Distributed{cell}` (real executor turn). The whole OS is one claim restated for pixels, input, and storage: every resource is an object-capability at some point on the distance parameter n, reached through ONE router, with the bounds the only thing that slides. n=1 (one machine) is first-class AND a stepping stone — the collapsed limit (immediate revoke, synchronous present, consistent capture), never a terminus. Nothing is invented that the firmament does not already prove; the desktop adds zero new trust and zero new authority model — it adds exactly one Target variant, one driver-PD per device seat, and a deliberately-tiny compositor multiplexer.  *(self-score 9/10)*

> A window is a dregg cell's surface capability; the desktop is the firmament's `(target,rights)` router extended with a third backing `Surface{cell}`, a third device-holding PD (the compositor), and a software-composited scene whose invariants are anti-ghost teeth — best-on-mac/linux NOW via a host-native rust-sel4 facade, seL4-native-verified LATER by the same source.

Models its scene as a verified dregg cell whose state is an ordered surface list; compositing is a present() turn gated by the SAME caveat machinery; renders the reserved trusted chrome (lineage label + X-ray dim) the client provably cannot paint.
7. L6 SHELL + WINDOW-MANAGER CELLS (untrusted) — a shell cell is a compositor-client that is ALSO a compositor to the apps it frames (Genode recursive stacking, free from cells-holding-caps-to-cells). Focus-routing within a subtree is delegated to it (Scenic hierarchical focus). It holds only the surface FACETS granted; it can be malicious without harming confined apps (CapDesk/DarpaBrowser).
8. L7 APP-PD CELLS + WEB SURFACES — apps are houyhnhnm cells; a window is a child Surface cell the app birthed via a surface FactoryDescriptor and granted (attenuated to a surface facet) to the shell/compositor. A WebView (servo, embeddable) is one such app-PD confined behind ONE surface-cap: a hostile page can paint its surface but reaches net/fs/clipboard/other-surfaces ONLY through caps it was granted — the DarpaBrowser confinement story, by construction.

**Semihost strategy**

The robigalia emulator is a host-native backend for the rust-sel4 API, drawn at TWO seams so ONE PD source tree runs on the host today and real seL4 unchanged. (A) `semihost-kernel`: PROMOTE dregg-firmament's LocalBacking (already a CNode slot-table with mint/revoke/is_attenuation) into a real EmulatedKernel by adding a synchronous Endpoint (Call parks until Recv — faithful seL4 synchrony) and a Notification (badge-OR + condvar wake). std-only, mac+linux, no privilege. (B) a `sel4-microkit`-API facade exporting protection_domain!/Handler/Channel/MessageInfo/memory_region_symbol!/debug_println with the EXACT upstream signatures, backed by (A): Channel::notify→Notification signal, Channel::pp_call→Endpoint Call, the run-loop blocks on the kernel's Recv exactly where seL4 Recv would block. The PD's init()+notified()/protected() bodies are LITERALLY the same text on both targets, differing only by a Cargo cfg. Precedent: this is UML's `um` arch port (same-code-targets-both at one well-defined seam) and gVisor's Sentry (a faithful re-impl of a privileged ABI in host userspace with a pluggable platform) — but dregg WINS over both because the executor is the verified Lean, so the emulator is a faithful MODEL not a from-scratch reimplementation. THE KEYSTONE PROOF (mirrors how the executor-state bridge + channels welds validated theirs): take dregg-pd/m0-hello + a 2-PD slice (PD-A notify's PD-B over a Channel; B's notified() fires) and run that SAME source against the facade under `cargo test` on macOS — no QEMU, no nightly, no build-std. Assert: B observed the badge, a pp_call round-trips a MessageInfo, a CNode mint+revoke enforces granted⊆held. Honest non-fidelity: host threads share an address space, so 'no ambient authority' is by-construction-in-the-API not MMU-enforced — the SAME gap UML's threads→SKAS faced, closed by the process-backed-PD upgrade (v1: shm_open/mmap regions) and the real-seL4 target; the emulator advertises Bounds::LOCAL and these are REAL (a host thread's revoke IS synchronous), so it is a faithful n=1 firmament, not a lossy mock. Caps are unforgeable via opaque typed handle + generation/epoch + kernel-side validity table (host RAM has no CHERI tag bits). The compositor's host backend: Surface{cell} present() resolves to a real present() turn + a host window-server blit; the SAME compositor-PD code, on seL4, blits a client frame-cap into the framebuffer region.

**Compositor model**

CAP-FIRST, NITPICKER-SHAPED, the compositor-PD as a third device-holding organ + the compositor modeled as a verified dregg cell. THE LOAD-BEARING MOVE: Target gains `Surface { cell: CellId }` (sel4/dregg-firmament/src/lib.rs), so a window = Capability{target: Surface(cell), rights} — the SAME struct, the SAME is_attenuation (granted⊆held) gate, the SAME introduce discipline, the SAME revoke. A SURFACE CELL is a real dregg cell whose verified state holds {geometry (size/pos/viewport-offset), frame-cap reference, owning lineage, client caption, stacking neighbor, contentDigest, sourceStateRoot, focusFlag, zLayer}. FIVE cap-confined operations, every verb a capability invocation not an ambient call: (1) CREATE-SURFACE — mints a surface cell + frame-cap, authorized ONLY by a surface-factory cap (EWS constructor/confinement = dregg FactoryDescriptor.allowed_cap_templates, real in cell/src/blueprint.rs; the contract BOUNDS which surface/input rights the cell may EVER hold — Arcan-style typed archetype). (2) PRESENT(surface, dirty-rect) — the client's frame dataspace is mapped in by an seL4 frame-cap grant (Qubes zero-copy / Nitpicker shared buffer, n=1 collapse); the compositor never copies, only maps+composites; requires a write/paint cap. (3) EMBED(parent, child) — a surface-tree edge authorized ONLY by Spec/Authority.lean apply_introduce (connectivity premise + both-consent + non-amplification) = Fuchsia's non-clonable token handshake as dregg's introduce; surface tree DECOUPLED from cell-ownership tree (RFC-0147). (4) GRANT-INPUT/GRANT-CAPTURE — delegating an attenuated receive-input/capture-region cap; this is the bit Wayland leaves as an ungoverned 'privileged client' — in dregg a global screenshot cap simply DOES NOT EXIST unless minted. (5) REVOKE — drops the surface cap (n=1: seL4_CNode_Revoke synchronous, the window goes dark instantly; n>1: the channels group-key epoch lift, #181). THE FACET IS A REAL, ALREADY-PROVEN MECHANISM: surface rights are CapabilityRef.allowed_effects (EffectMask, cell/src/facet.rs) with NEW bits 24-27 (EFFECT_PAINT/EFFECT_RECEIVE_INPUT/EFFECT_RESIZE_HINT/EFFECT_CAPTURE — bits 24-31 are FREE, highest used is bit 23) gated by is_facet_attenuation; a surface facet permits {paint, receive-input, resize-hint} and FORBIDS {seal, destroy, set-permissions, transfer}, so a compositor holding only a Viewport facet renders+routes-input but cannot tamper. TRUSTED PATH = unforgeable lineage label ∥ untrusted caption: the trusted part is DERIVED by the compositor-PD from the surface cell's CellId + authority lineage (which factory minted it, sovereign-vs-hosted, attenuation depth — all known to the executor, unforgeable by the client) — RICHER than Qubes' flat color border because dregg authority is a structured provenance lattice; rendered in reserved chrome at a zLayer no cell holds a cap to (Qubes 'outside client reach'), with X-ray/dim so a dark client is 'doomed to paint gray not white' (Nitpicker). INPUT = capability-routed + attested volition: events route only to the focused surface's input-cap holder; the compositor is the ROOT focus authority and ONLY the user selects focus (the input analogue of 'only connectivity begets connectivity'). When a gesture in a trusted surface authorizes a turn (cipherclerk 'sign this'), the compositor-PD ATTESTS it — emits a signed input-receipt the executor-PD can require as a turn premise (EWS user-volition, via dregg's existing attestation/receipt machinery, mirroring the net-PD's Ed25519 pre-check), so a value-moving turn can require proof a real user clicked. OUTPUT-INTEGRITY = unfoolability one hop out: model the scene invariants T1 non-overlap (granted⊆held on screen rectangles), T2 label-binding (label==owning cell's genuine lineage), T3 focus-exclusivity (≤1 focusFlag), T4 no-inference, as ANTI-GHOST TEETH via VerificationToolkit.AppSpec (an AppSpec whose admit-predicate is the conjunction) — app_commit_iff_admit gives 'a present commits IFF it respects the scene invariants', app_violation_rejected makes an overpaint/label-spoof/double-focus UNSAT, conservation/no-amplify re-exported. A Kill magic-key (compositor-PD-reserved) freezes layout + revokes a misbehaving surface's frame-cap (Nitpicker DoS brake); backing store is client-supplied (session quota) so DoS is structurally bounded; shared display surfaces are double-buffered (EWS covert-channel elimination, feeds info-flow pillar #31). GPU policy (the named open): the FIRST compositor does CPU composition of frame-caps (small TCB, the EWS/Nitpicker path); webgpu/servo acceleration is deferred to a separate UNTRUSTED render-PD whose output is itself a frame-cap, preserving the trust model.

**App / object model**

THE DESKTOP IS THE LEDGER; the compositor is a thin renderer that adds NO new trust. Four layers, each by SPECIALIZING a primitive that already exists (the WELD method). (1) APP = CELL (exists): every app is a dregg cell in the embedded World (starbridge-v2/src/world.rs); its authority is its c-list, its observable state its cell-forest, its history its receipt chain. (2) WINDOW = a CHILD CELL whose state holds a surface, addressed by a surface-capability — Fuchsia View/Viewport/ViewRef onto dregg with ZERO new kernel verbs: View := a surface-cell (born ZERO-balance + factory-validated, so it is a provenance-carrying object not an ambient handle); ViewRef := the surface-cell's CellId (already unforgeable, content-addressed via derive_raw, freely duplicable, cross-process — no new token type); Viewport := a CapabilityRef the parent holds reaching the child, attenuated to a surface FACET (EffectMask, narrow-only, proven non-vacuous by rejects_effect_mask_widening); the token-PAIR handshake := a GrantCapability turn whose granted⊆held gate IS Fuchsia's 'only matching-token holders link' — receipted, attenuable, revocable, strictly STRONGER than a kernel eventpair because provenance is on-chain. (3) THE VIEW TREE = THE OCAP GRAPH (just project it): the subgraph of surface-caps among cells, live in the World dynamics stream (CapabilityGranted/Revoked edges). FOCUS IS A CAPABILITY: the compositor-cell grants a short-lived, revocable receive-input facet to exactly one surface-cell; focus change = a revoke+grant turn, so 'only the focused view gets input' is ocap-enforced and the focus history is auditable on the receipt chain (WHY did this window get my keystroke). The compositor is a cell holding caps to framebuffer/input cells (Nitpicker as a cell); a window-manager is a compositor-client-cell that is ALSO a compositor to the apps it frames (recursive stacking, free). (4) THE DISTRIBUTED + WEB-FORWARD ANGLE = THE FIRMAMENT ROUTER, UNCHANGED (the payoff, nothing new in the cap layer): a surface-cap whose target is LOCAL renders directly (n=1, immediate); a surface-cell on a REMOTE federation is resolved via the executor→net path and the SAME window renders a remote cell's surface — the fluid reach-out applied to pixels. TWO transport modes: DETERMINISTIC content (retained scene-graph/DOM/verified app) ships STATE not pixels (the Croquet lesson — but dregg PROVES determinism where Croquet trusts it); the window subscribes to the remote surface-cell's state-commitment and the local compositor re-derives the pixels, and World::state_root makes the embedded remote surface SELF-ATTESTING (dregg's novel answer to Arcan's 'no visual identity to safely forward'). OPAQUE content (video/webgpu canvas) ships a content-commitment in state + bytes over the net-PD shared-ring, the commitment binding the bytes to the receipt chain. REACTIVITY: a SetField on a surface-cell's content/damage slot emits a SurfaceDamaged WorldEvent carrying the new content-commitment + damage rect; the compositor's existing per-frame since(cursor) poll re-renders — state-to-pixels is a pure function on the verified dynamics stream, so the desktop is fully REPLAYABLE + TIME-TRAVELABLE (starbridge-v2's replay/debug panels get surfaces for free). REVOCATION unifies via the firmament's two backings: seL4_CNode_Revoke locally (synchronous, window dark instantly) OR the group-key epoch lift distributedly (eventual) — 'stop sharing my screen' is a revoke turn with honest n-parametrized bounds. The two-tier throughput split (the named open): focus GRANTS are real receipted turns (rare, security-relevant, auditable); the per-event 1000Hz input STREAM flows on a faster cap-mediated shared-ring the receive-input facet authorizes, checkpointed into receipts at a cadence the proving-modality dial (#169) sets.

**Risks**

- GPU vs framebuffer TCB (Nitpicker's unsolved open problem, inherited): software CPU-composition keeps the compositor-PD tiny+verifiable (the first target), but webgpu/servo compositing drags a huge GPU driver into/beside the TCB. MITIGATION: the compositor does CPU blit of frame-caps; GPU acceleration is a SEPARATE untrusted render-PD whose output is itself a frame-cap the compositor composites — preserves the trust model, bounds performance. Decide this BEFORE adopting wgpu everywhere; the lockfile supports both Metal-direct and gpui_wgpu.
- The last-hop output-integrity binding (F1/F2): binding the scanned-out framebuffer to a cell's contentDigest needs either a verified display driver that hashes its scanout, OR trust in the GPU; an untrusted DMA-capable GPU can scribble arbitrary framebuffer regions unless an IOMMU cap confines it, and seL4 does not yet verify SystemMMU translation — so 'T1 non-overpaint holds against a malicious driver' is currently a NAMED ASSUMPTION (the graphics crypto-floor), not a theorem. MITIGATION: label it a severe-problem-with-a-closure-lane exactly like the crypto floor; the software-compositor T1-T4 are REAL now over honest cells; frame attestation is the frontier. NEVER launder this as solved.
- Input throughput as turns: routing every 1000Hz keystroke/mouse-move as a consensus-finalized turn is absurd. MITIGATION: the two-tier split — focus GRANTS are real receipted turns (rare, security-relevant); the per-event STREAM flows on a faster cap-mediated shared-ring the receive-input facet authorizes, checkpointed into receipts at the cadence the proving-modality dial (#169) sets. Must be pinned, or the 'every window op is a turn' claim either over-promises or tanks latency.
- Attested user-volition replay/confusion: making a turn REQUIRE a compositor-signed input-receipt is powerful but the receipt MUST commit to (surface CellId + gesture + nonce + which turn-field it authorizes), and the binding between 'this click' and 'this turn' must be pinned, or it is replayable. Risk: the compositor becomes a SECOND trusted authority over state. MITIGATION: the compositor holds only an attestation key and the executor checks a signature (mirroring net-PD Ed25519), never gaining state authority.
- Semihost fidelity gap honestly labeled: host threads share an address space, so 'no ambient authority' is API-by-construction, NOT MMU-enforced. Risk = silently treating the thread-model emulator as the security article. MITIGATION: advertise Bounds::LOCAL (which ARE real — host revoke IS synchronous) but label the isolation gap explicitly; close it via the process-backed-PD upgrade (shm_open/mmap) and the real-seL4 target — the SAME gap UML's threads→SKAS faced. Caps unforgeable via opaque handle+generation+validity-table (no host tag bits).
- Trusted-chrome real-estate spoofing (Nitpicker's orphaned-area + fullscreen-spoof): a malicious client could mimic a full trusted desktop if the reserved-chrome policy is loose. MITIGATION: at n=1 with a known app-PD set this is simpler — reserve a permanent lineage bar OR floating labels OR blank orphaned areas, but PIN the rule; the SAK (secure-attention key, the ONE non-cell piece held by the trusted PD at a zLayer no cell can reach) is what lets the human verify 'the trusted compositor, not an app, drew this label'.
- Executor-PD steady-state shape still open (FIRMAMENT §8.3): root-task-with-std + sel4-musl is experimental + root-task-only; the steady-state dregg.system wants the executor as a Microkit PD among many. Risk: Stage A as a root-task is the fast path but may not be the destination shape. MITIGATION: land root-task first (WALL step 4), evaluate the Microkit-PD musl substrate as a follow; the semihost de-risks by giving a real executor regardless.
- Remote (n>1) surface stale-frame leak: under n>1 revocation is eventual, so an in-flight frame may render AFTER a revoke ('screen-share stopped but one more frame leaked'). MITIGATION: gate remote surface-caps with the depth-parameterized non-revocation freshness scale (#150) so a revoked remote surface cannot paint even one stale frame, if the threat model demands it; otherwise label the eventual-revocation bound honestly in the n-table.
- Servo's heft for v1: servo drags servo-script/layout + SpiderMonkey + a large rc-quality dep tree. Risk: premature embedding. MITIGATION: the web-forward vision is real but staged — the SurfaceCell seam means a thinner web surface (WKWebView-class, or wgpu-canvas + minimal HTML) can fill the slot until full web fidelity is needed; servo plugs into the SAME OffscreenRenderingContext→surface-cell seam later.
