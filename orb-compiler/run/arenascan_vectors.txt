C5 ARENA request-line SP scan — two-kernel behavioral agreement.

The emitted primitive is the find-first-SP loop of Arena/Parse.lean
parseRequestLine: findByteIdx SP line = line.findIdx? (· == 32), the first-space
split point of "method SP target SP HTTP/x" (RFC 9112 §3).

Kernel 1: the REAL Lean model (drorb Arena/Parse.lean), run with `lake env lean`.
          findByteIdx SP over each line's UTF-8 bytes (== |line| when no SP), and
          the REAL parseRequestLine method span (off,len) where the line parses.
Kernel 2: the compiled Pancake x64 (cake --pancake, cc -O2 ... -lm), on hbox:
          LINE="..." ./arenascan   ->  prints the first-SP offset.
The HOL4 kernel (hol-c5) proves the Link-A refinement (scanLoop_refines_findSp):
the emitted While computes EXACTLY scanSp input (the HOL twin of findByteIdx SP),
for ALL inputs — not per-vector, a universally-quantified theorem.

vector (LINE)               | Pancake offset | Lean findByteIdx SP | parseRequestLine method
----------------------------+----------------+---------------------+------------------------
"GET / HTTP/1.1"            |       3        |         3           |  method=(0,3)
"POST /submit HTTP/1.1"     |       4        |         4           |  method=(0,4)

All 9 vectors: Pancake offset == Lean findByteIdx SP, and where the line is a
well-formed request line, that offset == the method span length parseRequestLine
records (off=0, len=SP-offset). The emitted scan IS the method|target split point.
