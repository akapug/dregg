## INSIGHTS TO PRESERVE

### Soundness / circuit rationale
- **Why no new circuit columns for side-table roots** — GROUP-4 (`state_commit = H4(inter1,inter2,inter3,ZERO)`) already absorbs `fields[0..7]`; assigning roots to field cells rides that absorption, so root-equality + the universe-A spec (`ActiveComponent.binds/encodes`) suffices — no per-effect in-circuit Merkle/set-membership gate needed (exception: nullifier non-membership, covered by the spend-proof PI cross-binding). (from: rebuild/_IR-EXTENSION-DESIGN.md)
- **Why no blanket Lean-replaces-Rust circuit delete** — Lean emits digest-EQUALITY statements; the Poseidon2 that PRODUCES the digest lives in the Rust AIRs (Lean's interpreter has no Hash node). Deleting the Rust AIRs deletes the only place digests bind to preimages ⇒ soundness loss. Keep them as diversity; only Rust-vs-Rust consolidation is safe (e.g. `note_spending_air` → `dsl::note_spending`). (from: rebuild/_RUST-CIRCUIT-CONSOLIDATION.md)
- **The abstract-spec ≠ concrete-row gap is REAL and load-bearing** — `CircuitSpecTriangle` proves over an abstract witness; the EffectVM emitter proves over the concrete 186-col layout; NO proof bridges them (the row's `state_commit = H4(...)` is not yet proven equal to the spec's `compressN(...)`). Group-B passthrough effects have WEAK row-intent (only "frozen except nonce") while their abstract specs are strong — honest amplification must STATE the weak row-intent and explicitly defer real semantics; never overclaim per-row for turn/accumulator properties (nullifier non-membership, queue FIFO, handoff membership are CROSS-AIR, conditional on "the PI root is genuine"). (from: rebuild/_EMITTER-AMPLIFICATION-WORKLIST.md)
- **Why LogUp wins** — bit-decomposition is O(bits) columns/wire; LogUp is one shared range-table + one aux + 3 constraints regardless of wire count/width. The nullifier bus also makes set semantics a clean cross-AIR algebraic statement (soundness clarity, not just perf). (from: rebuild/DESIGN-lookups-plonky3-perf.md)
- **Why plonky3-recursion (not pickles/Nova)** — only engine we can run (Rust, in-tree), STARK/AIR-shaped, ONE fixed verifier circuit to formally model. Silver (bundled trees) ships before Gold (recursive shrink); soundness does NOT hinge on reaching O(1) — each layer adds the SAME H2 public-input-binding obligation. (from: rebuild/DESIGN-recursion-aggregation-private-joint-turns.md)

### Authority / authorization / consistency rationale
- **Authorization is internalized, fail-closed on ANY leg** — `gateOK = credentialValidG ∧ capAuthorityG ∧ caveatsDischarged ∧ revocationGate` gates EXECUTOR admission; an out-of-band guarantee = a failure to internalize. (from: rebuild/_AUTHORIZATION-COMPLETE.md)
- **Authority-divergence verdicts (vs seL4)** — memory-access authority is a separate Blum-multiset model (a modeling choice, not a confusion); revocation is a state-operation + registry discipline (not an authority-edge); the only genuine conflation is async Notify (object modeled, authority missing). (from: rebuild/AUTHORITY-DIVERGENCE-FINDING.md)
- **Consistency-surface audit method** — 11 system-level trusted-assumption carriers, 10 non-trivial-proved (a discriminating witness per carrier); three by-design open obligations each isolated + given a realized/proved counterpart; ~15 crypto-standard carriers honestly Lean-trivial but never silent. The lone vacuity risk (`HolderAnonymity.ViewIndistinguishable`) was closed by `discriminatingAnon`. (from: rebuild/CONSISTENCY-SURFACE.md)
- **The dregg/dreggrs segregation is a SOURCE-OF-TRUTH boundary, not a language boundary** — dregg = Lean-truth (Rust is shadow/client); dreggrs = Rust-truth (self-contained heritage, backburner); enforced by code (`forest_is_root_agreeing` gate, `ProducerOutcome` enum), not prose. (from: rebuild/_DREGG-DREGGRS-MANIFEST.md)
- **Three disjoint things are all called "turn"** — protocol turn = `FullForestG` credentialed call-forest; `Exec.Turn` = a 4-field receipt-log row; commitment `Receipt` = Merkle+proof. A cell program is a PREDICATE filtering `(old,new)` pairs, not a declarative op-spec. (from: rebuild/_DREGG-ONTOLOGY-AND-PRODUCT.md)

### Federation / consensus / trust rationale
- **Federation = SSB + BFT finality + object-capabilities + double-spend safety** — SSB gives subjective trust, offline-first eventual consistency, feed identity; it LACKS finality, capabilities, double-spend protection (exactly dregg's surface). Strand = SSB feed extended: `StrandId := H_strand(creator_pubkey)` binds derivation, sequence-monotone + no-fork enforced at write. (from: rebuild/_FEDERATION-SSB-DESIGN.md)
- **CapTP mixed-trust boundary** — handoff certs + store-forward are trustless-when-proven (Ed25519/X25519); swiss table + GC + sessions are executor-trusted (collapses cleanly only at n=1; at n>1 it's a bounded assumption carried in verified `RecordKernelState`). Equivocation is content-independent (two incomparable blocks by one creator ⇒ evict + remove tip, keep both as evidence — the Byzantine tooth SSB lacks). (from: rebuild/_FEDERATION-SSB-ORIENTATION.md, _CAPTP-ORIENTATION.md)
- **Consensus paper groundings** — Sridhar et al. (arXiv 2411.01689) separate safety/liveness resilience (the asymmetric pair dominates Nakamoto for sleepy clients); Wong et al. (Matter Labs, Aug 2024) name the failure taxonomy (equivocation/reconfiguration/view-sync/DoS) — dregg's leaderless DAG sidesteps view-sync entirely. (from: rebuild/CONSENSUS-GROUNDING.md)
- **Polis autonomy is goodwill-independent** — a grantor hands a worker an attenuated, executor-enforced, revocable biscuit cap-token bound to the worker's OWN cell, never its keys; safety is the executor's law (`verify_token_authorization`), not politeness. Orchestration (embodiment/delegation/mandate/provenance/settlement) is µs–ms; the STARK (~0.5s) is a SEPARATE lane with different owners — don't couple them. (from: rebuild/_POLIS-SUBSTRATE.md, _PRODUCT-POLIS-ASSESSMENT.md)

### Intent / resource-theory rationale (and corrected overclaims)
- **Causal order is the only frame-invariant time** — happens-before is intrinsic; any total order picks a frame (wall-clock) and is conventional (Lamport 1978). `expiry: u64` is a fiction. A blocklace IS a causal set (discrete-gravity grounding: causal duration is COUNTED `#{z: a≺z≺b}`, never clocked). Commit-wait (Spanner TrueTime `[earliest,latest]`) converts a frame predicate into a causal fact by paying 2δ. Clock-sync bounds: n≥3f+1 unauthenticated, n>f authenticated — signatures buy resilience (dregg's posture: frame-authority attestations are signed). (from: rebuild/INTENT-REFS-time.md)
- **Justness is the unique non-trivial "typically warranted" liveness criterion (van Glabbeek)** — weak/strong/full fairness all score "−" (overcommitted ⇒ FALSE guarantees) AND fair schedulers provably CANNOT be implemented in CCS-like reactive languages (which dregg's executor IS). Justness is feasible + refinement/equivalence-stable (survives the Lean→Rust swap); weak fairness is not. The stutter-as-blocking-transition (`cellNextA` fail-close self-loop) is the transition justness excludes. (from: rebuild/INTENT-REFS-fairness.md)
- **Linearity IS the conservation defense** — no weakening + no contraction = consume exactly once (Girard); contraction IS double-spend. Propositions-as-sessions unifies demand⊣supply (cut = communication, cut-elimination = protocol running, Caires–Pfenning); session fidelity + deadlock-freedom become theorems not side-conditions. The turn-category must NOT be cartesian (no copy `Δ`, no erase `◇`) — that's how conservation is stated structurally. (from: rebuild/INTENT-REFS-linear.md, INTENT-REFS-resources.md)
- **Resource theory = SMC + convertibility preorder** (Coecke–Fritz); the two-layer split (rich SMC where morphisms live / thin collapsed preorder "any conversion exists?") is load-bearing — don't carry conservation on a thin posetal cat. Catalysis (`a⊁b` but `a+c⪰b+c`) is the formal model of a read-only/attenuating capability or a returned escrow bond. Decorated cospans (Fong) model open systems glued on boundaries; Petri nets are free SMCs (the cleanest token-conservation oracle). (from: rebuild/INTENT-REFS-resources.md)
