# Temporal logic in dregg: what exists, what is wired, what is missing

This document is the grounded answer to one question: *can you express temporal /
historical predicates over dregg — "X held until Y", "eventually Z", "this counter
stayed under k over the last N heights", a property over the whole receipt history —
and have it checked or enforced?*

It is written present-tense and grounded to `file:line` at HEAD. The short answer is
that dregg has **more temporal-logic machinery than usually remembered, but it lives
at three very different altitudes**, and only the narrowest of the three is a thing a
user can actually deploy. The interesting "historical predicate / running-range"
capability is **proven in Lean but not yet a writable guard or a query operator**.

## TL;DR verdict

| Capability | What it is | Status |
| --- | --- | --- |
| **Temporal-logic meta-theory over the real executor** | LTL (□/◇/◯), CTL (EX/AX/EU/AU/EF/AF/EG/AG), modal μ-calculus, justness/liveness — over the actual living-cell executor's turn trajectories | **ACHIEVED + grounded.** Proven, axiom-clean, wired to the real 46-effect executor. But it is a *proof/reasoning* surface, not a runtime query. |
| **Deployed, enforced temporal guard** | A cell caveat the executor + circuit actually check | **NARROW.** Only the height window `TemporalGate { not_before, not_after }` reaches the executor and the AIR. |
| **The "historical predicate / running-range" algebra** | `afterHeight` / `beforeHeight` / `withinWindow` / `cooledSince` / `rateBound` / `challengeWindow` / `untilEvent` / `sinceEvent` | **WIRED (STAGED).** A full, axiom-clean Lean algebra over real cell state. The register-reading members are now deployable caveats: `StateConstraint::{RateBound, CooledSince, UntilEvent, SinceEvent, ChallengeWindow}` — executor-enforced by the scalar evaluator (`cell/src/program/eval.rs`) and witnessed by the Effect-VM slot-caveat manifest (new tags 13–16, `circuit/src/effect_vm/{pi,verify}.rs`). STAGED behind the named *temporal-caveat verifier epoch* (old verifiers reject the new tags, so the rollout is lockstep); no existing cell declares them, so the deployed default is unchanged. |
| **Decidable flow / policy refinement** | `decideRefines : Flow → Flow → Bool`, the right-skewed simulation order | **PROVEN + WIRED** — but as a *deploy-time gate*, not a runtime predicate or a history query. |
| **Temporal queries over history** (`dregg-query`) | eventually/always/until/since over the receipt sequence | **NOT-SURVIVED.** `dregg-query` is conjunctive + safe-negation + aggregation with a freshness grade; it has zero temporal operators. Running aggregates exist but are point-in-time-at-frontier. |
| **`Pred` / derivative matching** | the caveat predicate algebra; Brzozowski/ERE matching | **NOT TEMPORAL.** `Pred` is boolean over a single state (or a single old→new transition). The derivative matcher is finite-string, not ω-regular/streams. |

So: *"we were supposed to be able to do full temporal-logic-type stuff"* — we **can,
and did, at the proof altitude** (LTL/CTL/μ-calculus over the real executor are
proven and grounded). What we **do not** have is a **usable surface**: there is no
writable historical-predicate caveat beyond the height window, and no temporal query
language. The "historical predicates and running-range thingies" you remember wanting
are exactly the `TemporalAlgebra` atoms — they are *built and proven*, just not
*exposed* as a guard or a query.

## 1. The temporal-logic meta-theory — ACHIEVED, grounded, proof-altitude

This is the part that genuinely answers "full temporal-logic stuff," and it is richer
than the flow/Büchi result alone. It is a proper LTL + CTL + μ-calculus tower built
**over the real living-cell executor**, not over an abstract toy Kripke structure.

- **LTL** — `metatheory/Dregg2/Proof/Temporal.lean`. Defines `Always` (□, line 72),
  `Eventually` (◇, line 77), `Next` (◯, line 83) over `trajA s sched : Nat →
  RecChainedState`, the trajectory generated by one executor step `cellNextA` of the
  real `execFullForestA`. Proves the introduction/distribution rules
  (`always_of_step_invariant`, `always_and`, `always_idem` S4) and, crucially, the
  **wired** OS theorems: `always_conserved` (per-asset conservation never drifts),
  `always_logMono` (audit log append-only), `always_revoked_persists` (revocation is
  permanent) at lines 305–327. `always_iff_reachable` ties □ to executor reachability.
  Non-vacuity via `#guard` on the real `transferCF`; `#assert_axioms`-clean.

- **CTL** — `metatheory/Dregg2/Proof/CTL.lean`. `EX/AX` (line 137), `EU/AU` (143),
  `EF/AF` (149), `EG/AG` (162) as lattice fixpoints, **grounded to
  `Temporal.livingSystem = inducedSystem livingCellA`** (line 249) — i.e. the actual
  executor. `livingAG_iff_temporalAlways` (385–411) bridges branching AG to linear □
  on every schedule; `livingAG_conserved` lifts conservation to the branching reading.
  Branching teeth (`EX_ne_AX`, `EF_ne_AF`, `AG_not_vacuous`) are real and decidable on
  a finite witness system. `#assert_axioms`-clean.

- **Modal μ-calculus** — `metatheory/Dregg2/Proof/MuCalculus.lean`. A positive-by-
  construction `Formula` with `mu`/`nu` fixpoints; proves each CTL operator IS the
  denotation of its standard μ/ν-formula (`encode_EF` = `μx. P ∨ ◇x`, `encode_AG` =
  `νx. P ∧ □x`, etc.). Alternation is non-decorative (`efPhi_ne_agComplPhi_denote`).

- **Justness / liveness** — `metatheory/Dregg2/Proof/Fairness.lean` +
  `CTLLiveness.lean`. van Glabbeek **justness** (not naive fairness) over the real
  executor: necessary-participant cells `npcA` and affected cells `afcA` read off the
  executor's lowering, `Commits = (execFullForestA …).isSome`. `just_progress` yields a
  genuine ◇ via a well-founded measure, with concrete teeth: a starvation schedule is
  *refuted* (`badSched_not_just`), and a fully-built liveness demonstrator
  (`refund_demo_eventually`) delivers an unconditional ◇ that a receipt eventually
  lands. `CTLLiveness.af_just_separates_plain` shows the just-path restriction is
  load-bearing, not vacuous. `#assert_axioms`-clean.

**What this altitude is and is not.** This is the assurance-case reasoning surface:
you *prove* "conservation holds AG over the real executor," "a refund eventually
settles on just schedules." It is the strongest temporal result in the tree and it is
honestly wired to the executor's dynamics. It is **not** a runtime mechanism — you do
not write one of these formulas into a cell and have a node check it per turn. It is
how *we* certify the protocol, not how a *user* writes a guard.

There is also a standalone, deliberately-abstract crypto piece —
`metatheory/Dregg2/Crypto/Temporal.lean` — that proves a range/window verifier
(`InWindow lo hi t`, sound + complete, dialed via `Dial.selective`) with **no** dregg
state wiring; it is the in-circuit verifier behind the deployed window gate (§2), not a
temporal logic over histories.

## 2. The deployed temporal guard — a height window, and only that

The single temporal thing a user can write into a cell today and have **enforced by
the executor and the circuit** is the two-sided height window:

- `cell/src/blueprint.rs` exposes `SimpleStateConstraint::TemporalGate { not_before,
  not_after }` (e.g. timelock vault at line 614, deadlines at 4199).
- `turn/src/executor/mod.rs:253` lowers `StateConstraint::TemporalGate { not_before,
  not_after }` into a `SlotCaveatEntry` with `type_tag = SLOT_CAVEAT_TAG_TEMPORAL_GATE`
  and the two heights as field params — so it is in-circuit (the AIR's slot-caveat
  check) and the verifier reads the executor's block height.
- There is also a witnessed `TemporalPredicate` / `Temporal` policy-authority kind
  (`executor/mod.rs:97`, `:325`) routed to the §8 verifier (`Crypto/Temporal`) — again
  a height window, just carried as a witnessed predicate rather than a native caveat.

The height clock is `s.log.length` (the receipt-chain length). So everything that is a
**pure function of height** is deployable today: `afterHeight h` = `TemporalGate {
not_before = h }`, `beforeHeight h` = `{ not_after = h }`, `withinWindow lo hi`, and
`cooledSince stagedAt period` = `{ not_before = stagedAt + period }` (this is how polis
cooling and the pre-rotation app get their cooldown). These are real, enforced, and
used.

What is **not** deployable: any temporal predicate that must **read a cell register**
to decide — see §3.

## 3. The historical-predicate / running-range algebra — PROVEN, not yet wired

This is the capability you were remembering. It exists, in full, as an axiom-clean Lean
algebra, in `metatheory/Dregg2/Authority/TemporalAlgebra.lean` and
`TemporalAlgebra2.lean`, with a causal companion in `Authority/CausalGuard.lean`.

The `TemporalAtom` family (TemporalAlgebra.lean ~111–145), each `eval`-able to `Bool`
against `(height, target-cell-pre-state)`:

- `afterHeight h`, `beforeHeight h`, `withinWindow lo hi` — the window members
  (these reduce to the deployed `TemporalGate`, §2).
- `cooledSince stagedAt period` — admit iff `stagedAt + period ≤ height`. A
  **running-range-since-an-event** gate (cooldown / vesting). Reduces to a one-sided
  window, so deployable as a `TemporalGate`.
- `rateBound counterField k` — admit iff the cell's counter register `< k`. A
  **running-rate** gate. **Needs a register read; not a pure height window.**
- `challengeWindow challengeField stagedAt period` — window elapsed AND challenge
  register `= 0`. The optimistic-rollup / fraud-proof shape. **Needs a register read.**
- `untilEvent flagField` (TemporalAlgebra2) — admit *while* a flag register is 0: the
  **U** operator over history. **Needs a register read.**
- `sinceEvent flagField` (TemporalAlgebra2) — admit *once* a flag register is set: the
  **S** operator. **Needs a register read.**

The algebra is genuinely welded to the real executor in Lean:
`temporalStateStepGuarded` installs the atom as a *precondition* above the existing
guarded write (the same shape as per-slot caveats), `temporalAtomsAdmit` is meet
semantics, and `committed_write_advances_clock` proves each committed guarded write is a
step of the height-clock transition system. The U/S pair is bridged to CTL's `EU`
fixpoint (`until_holds_EU_flip`). `CausalGuard.lean` adds blocklace-order causal guards
(`causallyAfter`, commit-reveal). All `#assert_axioms`-clean; non-vacuity by `#guard`
both polarities on concrete demo states.

**The wiring (staged).** The register-reading members are now writable, enforced
caveats — the plumbing the proofs were waiting for:

- `StateConstraint::RateBound { counter_index, k }` — "stayed under k" (`rateBound`).
- `StateConstraint::CooledSince { staged_at, period }` — "only after it cooled"
  (height-only; lowers to the deployed `TemporalGate` AIR teeth).
- `StateConstraint::UntilEvent { flag_index }` — "P until Y" (the U operator).
- `StateConstraint::SinceEvent { flag_index }` — "since the event" (the S operator).
- `StateConstraint::ChallengeWindow { challenge_index, staged_at, period }` — the
  optimistic-settlement gate (window elapsed AND no challenge filed).

Each reads the **committed pre-state register** (`old_state`; absent ⇒ `FIELD_ZERO`)
exactly as the Lean atom reads the target cell's pre-state record, honoring the proven
`temporalStateStepGuarded` / `temporalAtomsAdmit` semantics. They are **executor-enforced**
by the scalar evaluator (`cell/src/program/eval.rs`) and **circuit-witnessed** by the
Effect-VM slot-caveat manifest: the projection (`turn/src/executor/mod.rs::project_slot_caveat_manifest`)
emits new tags `SLOT_CAVEAT_TAG_{RATE_BOUND, UNTIL_EVENT, SINCE_EVENT, CHALLENGE_WINDOW}`
(13–16) that any proof consumer re-evaluates against the bound `state_before` slot view
(`circuit/src/effect_vm/verify.rs::verify_slot_caveat_manifest`). Teeth bite both
polarities in `cell/src/program/tests.rs` and `circuit/tests/state_constraint_air_teeth.rs`,
mirroring the Lean `#guard` non-vacuity examples value-for-value.

**Staged, not flipped.** This is a new caveat + verifier re-evaluation: the slot-caveat
manifest is carried in **public inputs** and enforced by an off-AIR verifier check, so the
AIR constraint polynomials (the VK bytes) are unchanged — but an old verifier rejects the
new tags as `unknown type_tag`, so deploying cells that USE these caveats requires every
verifier to upgrade in lockstep (the **temporal-caveat verifier epoch**). No existing cell
declares the new variants, so nothing flips by default; the epoch is the coordinate-with-
future-rollout step, deliberately deferred.

## 4. Flow / policy refinement — decidable, wired, but a deploy-gate not a query

`metatheory/Dregg2/Deos/FlowAlgebra.lean` + `FlowRefine.lean` give the result the
flow-algebra memory describes: dregg's flow composition is a **right-skewed Kleene
algebra with distributive meets** (`flow_choice_right_skewed`), and the refinement
order `A ≤ᶠ B` (online step-by-step simulation, no lookahead) is **decidable**:
`decideRefines : Proc → Proc → Bool`, sound + complete (`decideRefines_iff`), via the
σ-uniform simulation game `dupSim_iff_sim`. Because dregg flows are iteration-free,
`procSize` strictly decreases each move, every play is finite, and the Büchi acceptance
condition collapses to finite reachability — so this is the **decidable, PTIME-flavored
fragment** of the ω-regular/Büchi refinement story, not the full one. `#assert_all_clean`.

It is **wired** — but as a *deployment* gate, not a runtime predicate or a history
query. `dregg-deploy/src/refine.rs` treats a DreggDL plan's lowered turn-sequence as a
flow and runs two checks: **safe-upgrade** (`new ≤ᶠ old`) and **intent-conformance**
(`lowered ≤ᶠ intent`), routing `A ≤ᶠ B` through the exported Lean
`@[export] dregg_decide_refines` (`dregg-deploy/Cargo.toml:22`), with a Rust mirror
fallback. `metatheory/Polis/PolisFlowRefine.lean` reuses `decideRefines` as a
publicly-decidable political-capture floor (`flowCaptureBar`).

So refinement answers "does plan A behaviorally refine plan/intent B" at deploy time.
It does **not** answer "does this *history* satisfy temporal property φ" — `Flow` is a
spec object (a plan), not a stream of real receipts, and the gate compares two specs.

## 5. `dregg-query`, `Pred`, derivative matching — no temporal dimension

For completeness, the three places one might hope to find temporal queries, and why
they do not have them:

- **`dregg-query`** (`classify.rs` / `query.rs` / `fact.rs`) is a single conjunctive
  query + safe negation + aggregation (`Count/Sum/Min/Max`) over a closed 8-predicate
  EDB, with the CALM classifier grading each query **monotone** vs
  **finalized-dependent** and a `height_frontier` "fresh-as-of-height" annotation. That
  freshness grade and the per-fact height stamps are the *only* time-flavored elements;
  there is **no** eventually/always/until/since, and the evaluator is a point-in-time
  nested-loop join (`query.rs` ~351). The MMR (`mmr.rs`) attests the receipt slice is
  *complete* (non-omission) but that is completeness infrastructure, not a temporal
  operator. Running aggregates exist but compute one value at the frontier height.

- **`Pred`** (`metatheory/Dregg2/Exec/PredAlgebra.lean` ~127, mirrored in
  `cell/src/predicate.rs`) is boolean over a single state, or — via the reactive atoms
  `symUnchanged/symChanged/digUnchanged/digChanged` — over a single `(old, new)`
  transition. `Pred.eval : Pred → Value → Value → Bool` decides on two frames at once;
  there is no sequence dimension, no temporal connective.

- **Derivative / ERE matching** (`dfa/src/derivative.rs`, the boolean-matching reground)
  is Brzozowski derivatives over **finite** words (`matches(word: &[u8])`, `derives :
  List Value → … → Bool`). Finite-string / single-shot regular matching, **not**
  ω-regular over infinite streams — so not temporal-logic-capable as written.

## 6. The honest verdict and the path to a usable surface

**Verdict.** Split by altitude:

- *Temporal logic as proof / assurance*: **ACHIEVED + grounded.** LTL + CTL +
  μ-calculus + justness over the real executor, axiom-clean. This is real "full
  temporal-logic stuff," it just lives in the metatheory and certifies the protocol
  rather than running inside it.
- *Historical predicates / running ranges as a deployable guard*: **WIRED (STAGED).**
  The `TemporalAlgebra` atoms (cooledSince / rateBound / untilEvent / sinceEvent /
  challengeWindow) are now first-class `StateConstraint` variants, executor-enforced and
  circuit-witnessed (slot-caveat tags 13–16). Staged behind the temporal-caveat verifier
  epoch; no existing cell uses them, so the deployed default is unchanged.
- *Temporal queries over history*: **NOT-SURVIVED** as a feature — `dregg-query` never
  grew temporal operators; it stops at freshness-graded aggregation.

**What wiring would make temporal guards usable** (lowest-effort first, because the
Lean is already proven — this is plumbing, not invention):

1. **Lift the register-reading `TemporalAtom`s into `StateConstraint` + the AIR.**
   ✅ **DONE (staged).** `RateBound`, `CooledSince`, `UntilEvent`, `SinceEvent`,
   `ChallengeWindow` are first-class `StateConstraint` variants; each has a
   `SLOT_CAVEAT_TAG_*` (13–16; `CooledSince` lowers to the height-only `TemporalGate`
   tag) and a projection in `turn/src/executor/mod.rs::project_slot_caveat_manifest`
   beside the existing `TemporalGate` case, plus a re-evaluation arm in
   `verify_slot_caveat_manifest`. The soundness obligation was *already discharged* by
   `temporalStateStepGuarded` / `temporalAtomsAdmit` — the wiring inherits the proof.
   Deploying it is the named temporal-caveat verifier epoch (§3). Remaining: items 2–3.

2. **Give `dregg-query` history/window operators over the attested receipt slice.** The
   trace is already available and non-omission-attested (`mmr.rs`); only the operator
   vocabulary is missing. Add `Exists`/`Forall` over a height window (`∃h∈[lo,hi]. P`,
   `∀h∈[lo,hi]. P`), and running/windowed aggregates (sum-since-height,
   count-since-event), extending the CALM classifier to grade them (these are
   finalized-dependent). That makes "running-range thingies" *queryable* over real
   history, not just enforceable per-turn.

3. **(Optional, larger) Expose `decideRefines` as a policy/`Pred`-level "history
   conforms to flow F" check**, turning the deploy-time refinement gate into a runtime
   trace-conformance predicate. This is the path to checking that an *actual* receipt
   sequence refines an abstract intent flow, i.e. a genuine trace property witnessed
   on-chain rather than a spec-vs-spec deploy gate.

The through-line: the **proofs already exist** at every altitude. The missing thing is
exclusively the *exposure* — a `StateConstraint` variant + AIR tag for the register-
reading atoms, and a temporal operator vocabulary in `dregg-query`. Neither requires
new soundness theory; both are wiring against Lean that is already `#assert_axioms`-clean.
