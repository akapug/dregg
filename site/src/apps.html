Applications — dregg

Applications

Eight applications run on the dregg runtime today. Each is a real Rust crate (src/lib.rs with FactoryDescriptors and signed turn-builders), a README, and a passing test suite. Their core flows run end-to-end against the canonical TurnExecutor with the capability, mandate, and slot-caveat gates firing.

The defining property: none of these adds a domain-specific effect. There is no Effect::CastVote or Effect::ClaimBounty. Every rule is a generic dregg-native StateConstraint — WriteOnce, Monotonic, StrictMonotonic, FieldLteField, Immutable — minted onto a factory-born cell at birth. The executor enforces them on every turn; the app does no bookkeeping of its own. To walk any app — slots, caveats, turns, and a live try-it — open the use-case explorer.

Directory & identity

Names and credentials

starbridge-apps/nameservice

Nameservice

A federation name directory. register → set-target → renew → transfer → revoke. The name hash is WriteOnce (a name binds once), expiry is Monotonic (renewal only extends), and the revoked flag is WriteOnce (revocation is final).

WriteOnce Monotonic factory-born

starbridge-apps/identity

Identity

Issue credentials, present with selective disclosure, verify, revoke. The schema commitment is Immutable, issuance is a MonotonicSequence, the revocation root only grows, and only the authorized issuer (a public-root sender check) may write.

selective disclosure SenderAuthorized revocation
Coordination & markets

Pub-sub, voting, bounties

starbridge-apps/subscription

Subscription

A capability-gated bounded ring buffer. Grant publisher / consumer caps, publish, consume. Capacity and owner are Immutable; tail ≤ head always holds; publish advances the head as a monotone sequence, consume advances the tail.

ring buffer FieldLteField pub/sub

starbridge-apps/privacy-voting

Privacy Voting

Factory-born poll and ballot cells. The question hash is WriteOnce, tallies are Monotonic (a vote can never be erased by rewriting a tally downward), and each ballot's vote slot is WriteOnce — one vote per ballot, rejected by the executor on a second attempt.

one-vote-per-cell Monotonic tally blinded ballot

starbridge-apps/bounty-board

Bounty Board

A substrate-enforced state machine: OPEN(1) → CLAIMED(2) → SUBMITTED(3) → PAID(4). State is StrictMonotonic (no re-entry, no rewind); title and reward are WriteOnce; the claimant hash is WriteOnce, so the first claimer wins.

StrictMonotonic first-claimer-wins state machine
Governance & mandates

Route tables and bounded authority

starbridge-apps/governed-namespace

Governed Namespace

A DAO-controlled route table and service mesh. propose → vote (threshold) → commit → register. The governance committee root and threshold are Immutable; the table version is Monotonic; committing a table update advances the version as a monotone sequence.

threshold vote route table service mesh

starbridge-apps/compartment-workflow-mandate

Workflow Mandate

A DAG workflow under a bounded mandate. The step cursor is a MonotonicSequence that may never pass the charter terminal (cursor ≤ terminal); the commitment anchor is Immutable. Maps the Lean CompartmentWorkflowMandate theorems to Rust slot caveats and predicate-layer admission (clearance graph + spend policy).

clearance graph spend policy Lean-mapped

starbridge-apps/storage-gateway-mandate

Storage Gateway Mandate

A storage gateway under a volume ceiling. GET / PUT / LIST with prefix authorization and clearance labels; volume spent is Monotonic and may never exceed the ceiling (spent ≤ ceiling). Maps the Lean StorageGatewayMandate theorems to Rust caveats.

volume ceiling prefix auth Lean-mapped

Run them

From the repo or the browser

The devnet node seeds one of each factory-born app at genesis (node/src/starbridge_seed.rs), and the CLI drives them live. A rejected second claim or a shrinking tally is the caveat biting on the verified commit path — not a string compare in the app. 

# Privacy voting, live against a node
dregg voting open  --question "ship the gallery port?"
dregg voting tally --poll <id> --choice yes
dregg voting close --poll <id>
dregg voting show  --poll <id>

# Bounty board lifecycle (first-claimer-wins)
dregg bounty post   --title "port gallery" --reward 500
dregg bounty claim  --bounty <id>
dregg bounty submit --bounty <id> --uri ipfs://...
dregg bounty payout --bounty <id>

Or open the use-case explorer to walk each app's slots and caveats and try its turns against a node, no build required. Each app's web surface and tests live under starbridge-apps/<name>/.
