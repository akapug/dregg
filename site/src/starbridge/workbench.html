Starbridge Workbench — the inspector IDE
This is the Starbridge Workbench — the inspector IDE behind the shell. It boots a real in-browser wasm runtime — empty until you seed it (use Start here, below) — and renders every object as a dregg:// URI. The same inspectors back its three sibling surfaces: Explorer (browse a live devnet — cells, receipts with witness/proof status, receipt time travel per cell; offline it offers a loudly-labeled sample snapshot, never silent fakes) · Playground (drive the real in-browser wasm executor: stage a turn by verb, read the verified-Lean explanation before running, prove it with a real STARK, or submit the thin-HTTP fragment to a node) · Studio (author turns, predicates, and factories from generated-from-source catalogs — proof badges come from Lean's own axiom audit, never hand-set).
Live tonight: in-browser wasm runtime · remote read-only node connect · generated ontology/predicate/assurance catalogs · factory authoring + export. Not yet: browser blocklace sync (node HTTP only) · SDK signed-envelope submit from the browser (thin-HTTP fragment only).
